Operational Day Analyser (PFE)

Microservices
SpringBoot
Angular
MySQL
PrimeNG

The application should allow users to:

  • Upload Operational Day (OD) data from multiple sources.
  • Display the OD in clear formats to allow better understanding of the data.
  • Verify the data.
  • Compare different versions of OD.
  • Provide corrective scripts.

Development Highlights:

  • Développement de microservices principaux, dont la configuration, le discovery et l’API gateway.
  • Enhanced security by implementing role-based authorization and an authentication mechanism using JWT tokens.
  • Implemented XML and Excel parsers.
  • Optimized performance by implementing a low-complexity tree comparison algorithm, significantly reducing the number of operations required for large-scale comparisons.
  • Applied design patterns to enhance the scalability of the application such as Strategy and Observer.
  • The uses of Strategy pattern to support multiple data sources.
  • the use of Observer pattern to enhance the comparison algorithm.

L'application doit permettre aux utilisateurs de :

  • Importer les données de l'Operational Day (OD) à partir de plusieurs sources.
  • Afficher les OD dans des formats clairs pour une meilleure compréhension des données.
  • Vérifier les données.
  • Comparer différentes versions des OD.
  • Fournir des scripts correctifs.

Points forts du développement :

  • Développement de microservices principaux, dont la configuration, le discovery et l’API gateway.
  • Renforcement de la sécurité via l’authentification avec tokens JWT et l’autorisation basée sur les rôles.
  • Implémentation des parseurs XML et Excel.
  • Optimisation des performances en implémentant un algorithme de comparaison d’arbres à faible complexité, réduisant significativement le nombre d’opérations pour de grandes données.
  • Implémentation des patrons de conception pour améliorer la scalabilité de l'application, tels que Strategy et Observer.
  • Utilisation du patron Strategy pour gérer plusieurs types d’armes.
  • Utilisation du patron Strategy pour supporter plusieurs sources de données.
  • Utilisation du modèle Observer pour améliorer l'algorithme de comparaison.

Vehicle Maintenance Survey Dashboard

Tableau de bord de sondage sur l'entretien des véhicules

python
pandas
Dash

This project aims to create a dashboard that interprets the data from a survey regarding visits made to different types of vehicles.

Project Steps :

  • Selection of relevant survey questions. (You can se the survey question on the Survey)
  • Conversion of responses into actionable data stored in a CSV file.
  • Use of Pandas and Numpy for analyzing collected data.
  • Creation of visually informative graphs with Plotly.express and Plotly.graph_objs.
  • Organization of graphs using Dash, an interactive Python framework.
  • Improvement of presentation using dash_bootstrap_components.

Ce projet vise à créer un tableau de bord qui interprète les données d'une enquête concernant les visites effectuées sur différents types de véhicules.

Étapes du projet :

  • Sélection des questions d'enquête pertinentes. (Vous pouvez voir les questions de l'enquête sur le Survey)
  • Conversion des réponses en données exploitables stockées dans un fichier CSV.
  • Utilisation de Pandas et Numpy pour analyser les données collectées.
  • Création de graphiques informatifs visuellement avec Plotly.express et Plotly.graph_objs.
  • Organisation des graphiques à l'aide de Dash, un framework interactif en Python.
  • Amélioration de la présentation avec dash_bootstrap_components.
